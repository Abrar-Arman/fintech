Key Technical Decisions
LLM Processing: Backend Django endpoint handles Claude API calls for security, caching, and cost control. Frontend receives only matched services (no raw LLM output).

JSON Flexibility: PostgreSQL usage_inputs JSON field stores different schemas per pricing model type—no schema migrations needed when adding new models.

No Usage Storage: Cost estimates are calculated on-the-fly from user input; no need to store usage logs in MVP (can be added later for analytics).

Fuzzy Matching: Frontend-aware but backend-powered—LLM suggests, backend filters, frontend displays only valid matches.
